how scoring works
Each event gets two independent scores (0–100), shown on every card:
🍸 booze — an LLM reads the full event listing and estimates the probability of free drinks, citing its evidence.
logistics — starts at 50, then time-of-day, neighborhood, and RSVP friction adjust it (e.g. +25 weeknight sweet spot, −30 waitlisted).

Event-type preference isn't a score — it's the tags filter in the sidebar. Pick the tags you're interested in and only matching events show up (events with no tags at all are never hidden by it). No code to edit, and it can change as often as your taste does.

The badge's match number blends them: {% for name, w in scoring_info.weights.items() %}{{ name }} ×{{ w }}{{ ", " if not loop.last }}{% endfor %}. ≥{{ (scoring_info.confident * 100) | int }} ⇒ confident, ≥{{ (scoring_info.maybe * 100) | int }} ⇒ maybe, below ⇒ hidden.
